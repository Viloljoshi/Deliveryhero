03 / My Approach to Agentic Discovery at DH

How I'd build the discovery agent from day one.

The JD says "highly experimental and fast-moving." That's not a warning to me — it's the environment I've shipped AI in for five years. Here's how I'd approach building DH's agentic discovery experience: evaluation before agents, signals before queries, feedback loops before scale.

Phase 01
Define "Good" Before Building Agents
Measurement infrastructure first — always.
In non-deterministic AI systems, you can't measure what you haven't defined. Before writing a single agent prompt, I'd build the evaluation framework: conversational quality rubric (relevant → complete → delightful), a golden test dataset of real user sessions, and human + automated judge agreement rate. Ship an MVP, score it against the rubric, iterate. This is how you avoid "it looked good in the demo" failures at scale.
Metrics I'd own: eval score vs control baseline · judge agreement rate · hallucination frequency · task completion rate
Phase 02
Intent Layer — Signals Before Queries
The best discovery happens before the user types.
Most agents wait for the user's query. The best discovery experience starts with context signals: time of day, location, past order history, current session behavior, day-of-week patterns. I'd build an intent pre-classification layer that enriches any user query with structured context before the LLM sees it — dramatically improving response relevance without increasing latency. This is the "understanding hunger before the order" layer.
Metrics I'd own: context enrichment accuracy · pre-classification confidence distribution · latency budget per layer
Phase 03
Multi-Agent Architecture with Defined Scope
Specialized agents outperform generalist ones.
One agent trying to handle restaurant discovery, menu navigation, dietary filtering, and social recommendations will hallucinate across all four. I'd design specialized agents with defined scope and handoff protocols: a Restaurant Discovery Agent, a Dietary & Preference Agent, a Social/Trending Agent. Each with guardrails on what they can and cannot answer, and explicit fallback routing. The router decides which agent handles which intent class — not the agents themselves.
Metrics I'd own: agent routing accuracy · guardrail trigger rate · inter-agent handoff latency · scope violation frequency
Phase 04
Rapid Experiments — Ship Small, Measure Fast
An 80% solution in production beats a 100% solution in review.
I'd run tight experiment loops: narrow user cohort, specific intent class (e.g. "what should I eat tonight?"), 2-week cycle, clear hypothesis. Agent response quality vs control. Did the user complete the order? Did they accept the first suggestion or refine? Did they return within 48 hours? Each experiment produces a training signal for the next iteration. This is how you go from MVP to product-market fit on an agent that's never existed before.
Metrics I'd own: experiment cycle time · order completion rate (agent vs control) · session refinement rate · 48h return rate
Phase 05
Continuous Feedback Loops
Every interaction is a training signal — if you capture it.
Feedback collection can't be an afterthought. I'd instrument every agent interaction: did the user accept the recommendation? Refine it? Abandon? Click through to the restaurant? Order? Combine implicit signals (clicks, scrolls, time-to-decision) with lightweight explicit feedback (thumbs up/down on agent responses). Feed these back into the golden dataset and evaluation rubric. The agent that ships in month one should be substantially smarter by month three.
Metrics I'd own: feedback capture rate · implicit signal quality · eval score improvement per sprint · model improvement velocity
